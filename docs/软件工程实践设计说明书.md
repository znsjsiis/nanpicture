## 摘要

随着移动互联网和社交媒体的快速发展，图片分享已经成为人们日常生活中不可或缺的一部分。现有的图片分享平台存在用户体验不佳、功能单一、审核机制不完善等问题。针对当前现状，本文设计并实现了基于 Spring Boot 和 Vue 的图片分享系统。

系统后端采用 Spring Boot 框架，前端采用 Vue 3 框架和 TypeScript，数据库使用 MySQL 进行系统开发，分为公共模块、普通用户模块和管理员模块三大功能模块。公共模块包括用户注册登录、GitHub OAuth 第三方登录、文件上传等功能。普通用户模块实现图片的上传与分享、图片浏览、图片点赞、个人信息管理等核心功能。管理员模块提供用户管理、图片审核管理、数据统计等功能，实现对平台内容和用户的全面管控。

系统采用腾讯云 COS 对象存储服务实现图片的高效存储，使用 Redis 缓存提升系统性能，集成豆包 AI API 实现智能对话功能。通过该系统的使用，可以有效提高图片分享的效率和用户体验，为用户提供一个便捷、安全、智能的图片分享平台。

**关键词：** 图片分享；Spring Boot；Vue；对象存储；AI集成；权限管理

## Abstract

With the rapid development of mobile Internet and social media, picture sharing has become an indispensable part of people's daily life. Existing picture sharing platforms have problems such as poor user experience, single function, and imperfect review mechanism. In view of the current situation, this paper designs and implements a picture sharing system based on Spring Boot and Vue.

The system backend uses Spring Boot framework, frontend uses Vue 3 framework and TypeScript, and database uses MySQL for system development. It is divided into three functional modules: public module, ordinary user module and administrator module. The public module includes user registration and login, GitHub OAuth third-party login, file upload and other functions. The ordinary user module implements core functions such as picture upload and sharing, picture browsing, picture likes, and personal information management. The administrator module provides user management, picture review management, data statistics and other functions to achieve comprehensive control of platform content and users.

The system uses Tencent Cloud COS object storage service to achieve efficient storage of pictures, uses Redis cache to improve system performance, and integrates Doubao AI API to achieve intelligent dialogue functions. Through the use of this system, the efficiency of picture sharing and user experience can be effectively improved, providing users with a convenient, safe and intelligent picture sharing platform.

**Key words:** Picture Sharing; Spring Boot; Vue; Object Storage; AI Integration; Permission Management

## 1 绪论

### 1.1 引言

随着人们生活水平的日渐提高，越来越多的人选择通过图片来记录和分享生活。在互联网时代，图片分享已经成为社交媒体的重要组成部分。当前，图片分享平台数量庞大，但其中用户体验和功能设计却各有不同。由于缺乏有效的审核机制和智能交互功能，用户往往难以获得满意的分享体验。

"图片分享系统"是一个综合图片上传、浏览、点赞、审核等功能的综合性平台，为用户提供便捷的图片分享服务。通过注册账号，用户能够上传图片、浏览他人分享的内容、进行点赞互动等操作。管理员可以对用户上传的图片进行审核，确保平台内容的合规性，同时提供智能 AI 对话功能，提升用户交互体验。

### 1.2 研究目标与内容

**数据采集：** 支持用户通过文件上传或 URL 方式上传图片，系统自动获取图片元数据（尺寸、格式、体积等）。

**数据存储：** 使用腾讯云 COS 对象存储服务存储图片文件，MySQL 数据库存储图片元数据和用户信息。

**用户交互：** 实现图片点赞、评论、分类浏览等交互功能。

**智能集成：** 集成豆包 AI API，提供智能对话和知识库检索功能。

**权限管理：** 实现基于角色的权限控制，区分普通用户和管理员功能。

系统采用前后端分离的架构，以满足不同平台需求，并提高系统的可维护性和可扩展性。前端方面采用 Vue 3 框架和 TypeScript，使用组件化的开发模式。后端采用 Java 主流的开发框架 Spring Boot 2.7.6。为了权限与认证，使用自定义的权限拦截器。持久化框架使用 MyBatis Plus，通过 JDBC 访问 MySQL 数据库。为了管理 API 文档，使用 Knife4j 工具。缓存层使用 Redis 提升系统性能。

### 1.3 技术研究

#### 1.3.1 Spring Boot 技术研究

Spring Boot 是一个基于 Spring 框架的快速开发框架，它提供了一系列的自动化配置和约定优于配置的方式，可以帮助我们更快地搭建和开发应用程序。支持多种部署方式，如打包成 JAR 包，可以方便地部署到不同的环境中。本系统采用 Spring Boot 2.7.6 版本，结合 MyBatis Plus 实现数据持久化，使用 AOP 实现权限拦截。

#### 1.3.2 Vue 3 技术研究

Vue 3 是一款渐进式 JavaScript 框架，采用 Composition API 提供更灵活的代码组织方式。TypeScript 的引入增强了代码的类型安全性和可维护性。本系统前端采用 Vue 3 + TypeScript 开发，使用组件化架构实现页面的模块化开发。

#### 1.3.3 MySQL 技术研究

MySQL 是一款开源的关系型数据库管理系统，支持多种操作系统和编程语言。它提供了丰富的功能和机制，如数据存储、管理、查询等，能够满足各种场景下的需求。MySQL 支持多种数据存储引擎，如 InnoDB、MyISAM 等。本系统采用 MySQL 8.0 版本，使用 InnoDB 存储引擎支持事务处理。

#### 1.3.4 腾讯云 COS 技术研究

腾讯云对象存储（Cloud Object Storage，COS）是一种存储海量文件的分布式存储服务，具有高扩展性、低成本、可靠安全等特点。本系统使用腾讯云 COS 存储用户上传的图片文件，实现图片的高效存储和快速访问。

## 2 系统分析

### 2.1 可行性分析

#### 2.1.1 经济可行性

系统开发中使用的开发工具 IntelliJ IDEA、VS Code 的社区版本都是免费的，腾讯云 COS 提供免费额度满足开发测试需求。整个开发过程可以在自己的电脑上进行开发，不用租赁服务器来支撑开发所需的运行环境要求，因此这个项目的开发经济是可行的。

#### 2.1.2 技术可行性

本系统后端采用 Spring Boot 2.7.6 框架，这是一款成熟稳定的 Java 开发框架。前端采用 Vue 3 和 TypeScript，具有长时间存在、稳定性高的特点。在持久层框架方面，采用了 MyBatis Plus 框架，这一选择保证了高度的可维护性。数据库方面选用了 MySQL，这是一款广泛应用的关系型数据库。采用了 Redis 作为高性能的缓存数据库，有效提升了系统的响应性能。对象存储使用腾讯云 COS，提供稳定的文件存储服务。综上所述，该项目在技术上具备了可行性。

#### 2.1.3 管理可行性

这个系统采用前后端分离架构，分为普通用户和管理员两种角色，普通用户和管理员的功能各不相同。普通用户可以进行图片上传、浏览、点赞等操作，管理员在后台可以对用户信息、图片审核、数据统计等进行管理，因此这个系统在管理上是可行的。

### 2.2 需求分析

#### 2.2.1 公共模块需求

**用户注册：** 用户在进入注册页面时，在注册框中输入账号、密码、昵称等用户信息，填写完毕点击注册按钮；如果用户信息正确，那么系统会提示注册成功并跳转到登录页面。

**用户登录：** 普通用户可以进入到系统前台展示页面进行图片浏览和上传；管理员用户进入后台管理系统，进行相关信息的管理。

**GitHub OAuth 登录：** 支持用户通过 GitHub 账号进行第三方登录，简化注册流程。

**文件上传：** 支持用户通过文件或 URL 方式上传图片到腾讯云 COS。

#### 2.2.2 普通用户需求

**图片上传：** 普通用户可以上传图片，支持单张上传和批量上传，上传时可以填写图片名称、简介、分类、标签等信息。

**图片浏览：** 普通用户可以浏览系统内的所有图片，支持按分类、标签进行筛选，支持搜索功能。

**图片点赞：** 普通用户可以对喜欢的图片进行点赞操作。

**个人信息管理：** 普通用户可以修改个人信息，包括头像、昵称、简介等。

**AI 对话：** 普通用户可以使用豆包 AI 进行智能对话。

#### 2.2.3 管理员用户需求

**用户管理：** 用户管理界面展示了所有注册用户的相关信息，包括用户 ID、账号、昵称、头像等，管理员可以进行用户的查询和删除操作。

**图片审核管理：** 管理员可以查看所有图片的审核状态，对待审核的图片进行审核操作（通过/拒绝）。

**图片管理：** 管理员可以查看所有图片信息，进行编辑和删除操作。

## 4 系统实现

### 4.1 公共模块实现

#### 4.1.1 用户注册登录模块

**(1) 用户注册**

用户在进入注册页面时，在注册框中输入账号、密码、确认密码等用户信息。系统对账号进行唯一性校验，对密码进行加密存储。注册成功后跳转到登录页面。

**(2) 用户登录**

用户在进入登录页时，在登录框中输入账号、密码。系统对用户提交的账号和密码进行验证，验证成功后根据用户角色判断跳转到前台首页或后台管理页面。

**(3) GitHub OAuth 登录**

系统支持 GitHub 第三方登录，通过 JustAuth 库实现 OAuth 2.0 认证流程。用户点击 GitHub 登录按钮后，跳转到 GitHub 授权页面，授权成功后系统获取用户信息并完成登录。

#### 4.1.2 文件上传模块

系统支持两种图片上传方式：

**(1) 文件上传：** 用户选择本地图片文件上传，系统将文件上传到腾讯云 COS，并自动获取图片的元数据（尺寸、格式、体积等）。

**(2) URL 上传：** 用户输入图片 URL，系统从 URL 下载图片并上传到 COS，同样获取图片元数据。

**(3) 批量上传：** 管理员可以通过搜索关键词批量从图片网站抓取图片并上传。

#### 4.1.3 API 文档模块

系统使用 Knife4j 生成 API 文档，通过 Swagger 注解标记接口信息，方便前端开发和接口测试。

### 4.2 面向普通用户的功能实现

#### 4.2.1 图片上传模块

用户可以上传图片，上传时填写图片名称、简介、分类、标签等信息。系统自动获取图片的元数据并存储到数据库。上传的图片默认进入待审核状态，需要管理员审核通过后才可公开显示。

#### 4.2.2 图片浏览模块

**(1) 首页展示：** 首页展示已审核通过的图片列表，支持分页浏览。

**(2) 分类筛选：** 用户可以按分类筛选图片。

**(3) 标签筛选：** 用户可以按标签筛选图片。

**(4) 搜索功能：** 用户可以通过关键词搜索图片名称和简介。

#### 4.2.3 图片详情模块

图片详情页面展示图片的完整信息，包括图片名称、简介、分类、标签、上传者、上传时间、点赞数等。用户可以在详情页进行点赞操作。

#### 4.2.4 图片点赞模块

用户可以对喜欢的图片进行点赞操作。点赞信息存储在 `picture_like` 表中，一个用户对一张图片只能点赞一次。点赞后图片的点赞数自动增加。

#### 4.2.5 个人信息管理模块

用户可以修改个人信息，包括头像、昵称、简介等。头像修改通过文件上传实现。

#### 4.2.6 AI 对话模块

系统集成豆包 AI API，用户可以在专门的对话页面进行智能对话。支持知识库检索功能，可以通过上传 Markdown 文件构建知识库。

### 4.3 面向管理员的功能实现

#### 4.3.1 用户管理模块

后台用户管理界面展示了所有注册用户的相关信息，包括用户 ID、账号、昵称、头像、角色等。管理员可以查询用户信息，对违规用户进行删除操作（逻辑删除）。

#### 4.3.2 图片审核管理模块

管理员可以查看所有图片的审核状态。待审核的图片显示在审核列表中，管理员可以进行审核操作：

- **通过审核：** 图片状态变为已通过，可以在前台展示
- **拒绝审核：** 图片状态变为已拒绝，填写拒绝原因

#### 4.3.3 图片管理模块

管理员可以查看所有图片信息，包括审核通过的、待审核的、已拒绝的。管理员可以编辑图片信息或删除图片。

## 6 总结与展望

基于 Spring Boot 和 Vue 的图片分享系统的设计与实现，总体框架分为三个部分：公共模块、普通用户模块和管理员模块。公共模块实现了用户注册登录、GitHub OAuth 第三方登录、文件上传等功能。普通用户模块实现了图片上传、图片浏览、图片点赞、个人信息管理、AI 对话等功能。管理员模块实现了用户管理、图片审核管理、图片管理等功能。

系统采用前后端分离架构，后端使用 Spring Boot 2.7.6 框架，前端使用 Vue 3 和 TypeScript。数据库采用 MySQL，缓存使用 Redis，文件存储使用腾讯云 COS 对象存储服务。通过 MyBatis Plus 实现数据持久化，使用 Knife4j 生成 API 文档，使用 JustAuth 实现 GitHub OAuth 登录。集成豆包 AI API 实现智能对话功能。

本系统基本实现了预期的功能。用户可以通过系统方便地分享图片、浏览他人分享的内容、进行点赞互动。管理员可以对平台内容和用户进行有效管理，确保平台内容的合规性。

由于首次接触某些技术框架，在开发过程中遇到了一些问题。例如，在对象存储配置、权限拦截器设计等方面需要不断摸索和学习。在批量图片上传功能中，遇到了网络连接问题，需要进一步优化网络请求配置。前端页面设计的美感还有待提升。

总之，图片分享系统还有很多不足需要优化。未来在不断学习相关知识时，将不断完善系统的功能，包括增加图片评论功能、优化图片搜索算法、增加数据可视化分析等功能，为用户提供更好的图片分享体验。
