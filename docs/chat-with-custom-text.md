# 使用自定义文本进行问答（无需创建知识库）

## 🎯 功能说明

这是一个**轻量级**的知识问答方案，无需在火山引擎控制台创建知识库，直接提供文本即可让 AI 基于你的内容回答问题。

### 适用场景

✅ **临时性查询** - 快速基于一篇文档问答  
✅ **小型文本** - 文本长度在几千字以内  
✅ **测试验证** - 测试知识库效果  
✅ **一次性使用** - 不需要长期维护的知识

❌ **大型文档** - 超过 10,000 字的文档（建议使用正式知识库）  
❌ **长期使用** - 需要反复使用的知识（建议创建正式知识库）  
❌ **多文档管理** - 需要管理多个文档（建议使用知识库功能）

## 🚀 快速使用

### 方式 1：通过 HTTP 接口调用

#### 基本用法

```bash
POST http://localhost:8123/api/doubao/chatWithText
Content-Type: application/x-www-form-urlencoded

question=你的问题&contextText=你的背景文本
```

#### 示例 1：公司产品介绍

```bash
curl -X POST "http://localhost:8123/api/doubao/chatWithText" \
  -d "question=公司的主要产品有哪些？" \
  -d "contextText=我们公司成立于 2020 年，专注于人工智能领域。
主要产品包括：
1. 智能客服系统 - 为企业提供 24 小时在线客服服务
2. 数据分析平台 - 帮助企业分析业务数据，提供决策支持
3. 图像识别系统 - 用于安防、医疗等领域的图像自动识别
我们的客户遍布全国，已有超过 500 家企业使用我们的产品和服务。"
```

#### 示例 2：规章制度问答

```bash

  -d "question=年假怎么计算？" \
  -d "contextText=公司员工手册 - 休假制度

第三章 年假规定

第五条 员工享受年假的条件：
```

### 方式 2：在代码中使用

```java
@RestController
@RequestMapping("/example")
public class ExampleController {
    
    @Resource
    private DoubaoService doubaoService;
    
    @GetMapping("/qa")
    public String questionAnswer() {
        String context = "我们的产品包括：手机 A1（价格 2999 元）、平板 B2（价格 3999 元）、笔记本 C3（价格 5999 元）";
        String question = "平板多少钱？";
        
        return doubaoService.chat(question, 
            "请根据以下信息回答：" + context);
    }
}
```

## 🔧 参数说明

### 请求参数

| 参数             | 必填  | 说明        | 示例           |
|----------------|-----|-----------|--------------|
| `question`     | ✅ 是 | 用户的问题     | `年假怎么计算？`    |
| `contextText`  | ✅ 是 | 背景文本/参考资料 | `公司规章制度...`  |
| `systemPrompt` | ❌ 否 | 额外的系统提示词  | `你是一位 HR 专家` |

### 返回结果

```json
{
  "code": 0,
  "data": "根据提供的信息，年假计算方式如下：...",
  "message": "success",
  "info": "使用自定义文本作为背景知识"
}
```

## 📊 与正式知识库的对比

| 特性       | 自定义文本        | 正式知识库       |
|----------|--------------|-------------|
| **使用门槛** | 低，直接使用       | 需要创建知识库     |
| **响应速度** | 快，即时使用       | 较慢，需检索      |
| **文本长度** | 适合短文本（<10KB） | 支持长文本（>1MB） |
| **持久化**  | 不保存，每次提供     | 永久保存        |
| **成本**   | 按 tokens 计费  | 存储免费，检索收费   |
| **准确度**  | 较高           | 最高          |
| **适用场景** | 临时查询         | 长期使用        |

## ⚠️ 注意事项

### 1. 文本长度限制

- **推荐长度**：500-5000 字
- **最大长度**：不超过 8000 tokens（约 1 万汉字）
- **过长处理**：分段提供或创建正式知识库

### 2. 文本格式建议

✅ **结构化** - 使用标题、段落、列表  
✅ **清晰明确** - 避免歧义表述  
✅ **重点突出** - 关键信息放在前面  
✅ **逻辑连贯** - 保持内容的连贯性

❌ **杂乱无章** - 缺乏组织  
❌ **过于简略** - 缺少必要信息  
❌ **矛盾冲突** - 自相矛盾的表述

### 3. 提高准确性技巧

**技巧 1：提供完整上下文**

```
不好的例子：
contextText=产品价格 2999 元

好的例子：
contextText=iPhone 15 基础版，128GB 存储，黑色，产品价格 2999 元
```

**技巧 2：使用明确的表述**

```
不好的例子：
contextText=可能、大概、左右

好的例子：
contextText=确切数字：2999 元、5 天、10%
```

**技巧 3：添加示例**

```
contextText=报销流程：
1. 填写报销单
2. 部门经理审批
3. 财务审核
例如：张三报销差旅费 500 元，按此流程执行
```

## 🐛 常见问题

### Q1: 为什么 AI 的回答不准确？

**A:** 可能的原因：

1. 背景文本中没有包含足够的信息
2. 文本表述不够清晰
3. 问题太宽泛

**解决方法：**

- 补充更详细的背景信息
- 重新组织文本结构
- 提出更具体的问题

### Q2: 文本太长怎么办？

**A:**

1. 精简文本，只保留关键信息
2. 分段提供，多次对话
3. 创建正式知识库

### Q3: 如何让 AI 只根据文本回答？

**A:** 在 systemPrompt 中明确要求：

```
systemPrompt=请严格根据提供的背景知识回答，不要使用你的训练数据。如果背景知识中没有答案，请直接说不知道。
```

### Q4: 可以上传文件吗？

**A:** 当前接口只接受文本。如果需要处理文件：

1. 手动复制文件内容到 contextText
2. 使用知识库功能（支持 PDF、Word 等格式）

## 📝 最佳实践

### 1. 文本组织模板

```markdown
【主题】明确的主题名称

【基本信息】
- 时间：相关时间
- 地点：相关地点
- 人物：相关人员

【核心内容】
分点列出关键信息

【示例】（可选）
具体例子帮助理解

【注意事项】
需要特别说明的内容
```

### 2. 问题设计技巧

✅ **具体问题** - "年假几天？"而不是"休假制度？"  
✅ **单一问题** - 一次问一个问题  
✅ **明确对象** - "iPhone 15 的价格"而不是"价格"

### 3. 迭代优化

第一次回答不满意时：

1. 补充更多背景信息
2. 重新表述问题
3. 添加约束条件
